# 贡献说明

感谢你关注这个仓库。

这个工程的目标不是堆叠页面数量，而是把 HarmonyOS 中真正可验证的 FAST Kit 能力接入、展示并解释清楚。因此，提交代码时请优先保证“文档语义、页面行为、Native 结果”三者一致。

## 提交前建议

1. 先阅读 `README.md`，确认当前工程范围和限制。
2. 若修改 `RectPartition`，请同时检查：
   - 输入坐标文本
   - 原始区域画布
   - 优化结果坐标
   - 优化结果画布
   - 结果来源说明
3. 若修改 `SegmentMap`，请同时检查：
   - 输入数组
   - 更新前查询结果
   - 更新后查询结果
   - 页面数组预览
4. 若修改 Native 桥接层，请同步检查 ArkTS 调用参数、错误提示和 README 说明。

## 请不要提交

- `build-profile.json5`
- 本地签名证书、密码、profile
- `.hvigor/`
- `.home/`
- `oh_modules/`
- `entry/build/`
- `entry/.cxx/`
- 本地调试日志

## 推荐修改方式

- 优先补充中文注释，帮助后续开发者快速理解代码。
- 优先保持页面文案与实际功能边界一致，不要伪造“已支持”的能力。
- 如果当前 SDK 或设备环境不支持某项能力，请明确写出限制，而不是隐藏问题。

## 验证建议

- 至少做一次页面级手动检查。
- 若条件允许，使用模拟器或真机重新运行相关页面。
- 如遇 Native 调用失败，请结合 `hilog` 和环境状态文案一起排查。
